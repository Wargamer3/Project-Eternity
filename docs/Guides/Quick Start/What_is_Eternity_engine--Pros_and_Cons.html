Pros and Cons


Pros


• It is entirely free and open source and is specialized in SRW gameplay. Some other engines like SRPG studio are designed around Fire Emblem gameplay. The only proper SRW engine is SRC but is not in english.
• It has many built-in precoded features from SRW from various eras. Squad system, terrain bonuses, barriers, MAP attacks, multi-units and others.
• Being an open source toolbox, it allows for significant flexibility with extensions by just dropping new dlls. You can add new AI scripts, skill effects, unit types, cutscene actions, etc. You can also just create entirely new mods that reuse those tools to make something like a shooter, and advance war clone, etc.
• It is very light-weight, able to run on computers as old as Windows 7, and file sizes are very small allowing for easy distribution. Could be ported back to XP if needed.
• It is meant to be a community effort so it has all the content of SRWE available free to use. Sprites, animations, tiletsets, AI, etc.

Cons


• Being based on XNA, you need some workaround to make it work with visual studio if you want to modify the code. It will also require you to compile sprites to use them. (Can be worked around with TEdit)
• With the exception of units and characters, games are constructed entirely through the use editors so you can’t easily edit assets with a text editor.
• The engine was initially designed by someone with no knowledge of SRW and was inspired by Armored Core and Unreal Tournament, making some editors include non standard SRW features which can be confusing.
• At this moment, the Community is very small.
